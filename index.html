Xebium

Combine the power of Selenium and FitNesse and get more out of both!

Integration testing becomes easier and more fun with Xebium. Selenium is a great tool for regression testing, and FitNesse is great for maintainable acceptance testing. Combining them gives you the ease of FitNesse and the power of Selenium, in a non-intrusive way: record your Selenium tests with the SeleniumIDE like you're used to, translate them automatically and run them from FitNesse!

✔  Full Selenium-IDE - FitNesse roundtrip with your web tests.

✔  Create data-driven tests.

✔  Tests are executed using the modern WebDriver interfaces.

✔  Tests are run from FitNesse using the SLIM engine.

Free Download

You can also clone the project with Git by running: 

$ git clone git://github.com/xebia/Xebium

Two way translation

With use of the Xebium Format a script recorded in Seleinium IDE is automatically translated into wiki syntax and can be pasted into FitNesse. And reverse: Wiki syntax can be copied back to Selenium IDE and can run as a selenium script. Very useful for debugging

Power of FitNesse

You can execute any custom build fixture code around the execution of the selenium test and use the default Setup and Teardown mechanism. This is great for bringing the system under test into the right state, setting up pre-conditions and cleaning up before running a test.


Improved maintainability

Because the tests are part of the FitNesse wiki you can create pages and test suites. These can be grouped in a tree structure, just like normal FitNesse tests. With FitNesse scenario table fixtures you can easily reduce duplication by making reusable components. Finally the documentation can be added right with the tests in the wiki. All this makes the tests much more maintainable.

Data driven testing

FitNesse's variable substitution makes tests less dependent on hardcoded values.
Use scenario tables to out-of-the-box create new methods enabling data-driven testing.

Reduced development costs

Xebium empowers you to use FitNesse without any development support, no coding is required to get you started.

 

Quickstart (for developers)

  • Prerequisites: 
    For retrieving the project from Github Git is required
    For building the project Maven is required
    The following commands should be executed from a commandline terminal
  • clone/fork the github project by running: 
    $ git clone git://github.com/xebia/Xebium
  • Now build the project and start the Fitnesse server from the projects root folder: 
    $ mvn -Pfitnesse test
  • After these two steps the application should be succesfully started, showing something like:
    • FitNesse (v20110104) Started...
    [java] 	port:              8001
    [java] 	root page:         fitnesse.wiki.FileSystemPage at ./FitNesseRoot
    [java] 	page version expiration set to 0 days.
  • Open Firefox to enter the Fitnesse Wiki
    • http://localhost:8000/

    You are ready to go!

Getting started

  • Prerequisites:
    • Make sure both FitNesse and Firefox with the Selenium IDE plugin are installed.
    [IMPORTANT]
    Please be aware of the fact that the installation below depends on a previously released version of Xebium.
    In the meantime development on Xebium has continued, which has led to new features and broader browser (version) support. The 0.3 version of Xebium-jar-with-dependencies best functions with Firefox 4.0.x and does not support any Firefox version higher than 4.x.
    For the most up-to-date version of Xebium, please follow the Quickstart for Developers (described to the right), which results in a fully setup Fitnesse installation, including the latest version of Xebium and detailed examples of it's capabilities
  • Save the Xebium-jar-with-dependencies.jar file to the same location as fitnesse.jar
  • Start FitNesse
  • edit the root page of your tests; insert: 
    
!define TEST_SYSTEM {slim}
!path xebium-0.3-jar-with-dependencies.jar
  • Create a testpage (nested under the the rootpage and perhaps within a testSuite).
  • Edit it, add the following import: 
    
| Import |
| com.xebia.incubator.xebium |
  • Open Firefox and add the Selenium-Xebium-formatter from the following location:
    selenium-xebium-formatter
  • Follow the installation steps, use the Open with option and select Firefox for applicaiton, when finished restart Firefox
  • When using SeleniumIDE 1.0.11 (or higher) goto Options -> Options... and check the "enable experimental features" checkbox
  • Goto Options -> Format and select the xebium-formatter from the list of formats.
  • Now go ahead and record any test in seleniumIDE. When a test is finished, click the Source tab and copy its content into the copy buffer.
  • Go back to to the previously created testpage in FitNesse and paste the recorded script from the buffer.
  • Save it, and run the FitNesse test.
