Xebium

Combine the power of Selenium and FitNesse and get more out of both!

Integration testing becomes easier and more fun with Xebium. Selenium is a great tool for regression testing, and FitNesse is great for maintainable acceptance testing. Combining them gives you the ease of FitNesse and the power of Selenium, in a non-intrusive way: record your Selenium tests with the SeleniumIDE like you're used to, then translate them automatically and run them from FitNesse!

✔  Full Selenium-IDE - FitNesse roundtrip with your web tests.

✔  Create data-driven tests.

✔  Tests are executed using the modern WebDriver interfaces.

✔  Tests are run from FitNesse using the SLIM engine.

Free Download

You can also clone the project with Git by running: 

$ git clone git://github.com/xebia/Xebium

Two way translation

A script recorded in selenium can be easily translated into wiki syntax and pasted into FitNesse. The other way around works too: wiki syntax can be copied to SeleniumIDE and translated into a selenium script. Very useful for debugging

Power of FitNesse

You can execute any custom build fixture code around the execution of the selenium test. This is great for bringing the system under test into the right state for the test to run. It is also easier to clean up after a test.


Improved maintainability

Because the tests are part of the FitNesse wiki you can create pages and test suites. These can be grouped in a tree structure, just like normal FitNesse tests. With FitNesse scenario table fixtures you can easily reduce duplication by making reusable components. Finally the documentation can be added right with the tests in the wiki. All this makes the tests much more maintainable.

Data driven testing

Use FitNesse's variable substitution to make tests less dependent on hardcoded values.

 

Quickstart (for developers)

  • clone/fork the github project by running: 
    $ git clone git://github.com/xebia/Xebium
  • Execute: 
    $ mvn -Pfitnesse test

Getting started

Prerequisites

Make sure both FitNesse and Selenium are installed.

  • Save the Xebium-jar-with-dependencies.jar file to the same location as fitnesse.jar
  • Start FitNesse
  • edit the root page of your tests; insert: 
    !define TEST_SYSTEM {slim}
!path Xebium-1.0-SNAPSHOT-jar-with-dependencies.jar
  • Create a testpage (nested under the the rootpage and perhaps within a testSuite).
  • Edit it, add the following import: 
    | Import |
| com.xebia.incubator.xebium |
  • Open SeleniumIDE and go to Option -> Options.
    Click the Formats tab and click the Add button.
  • Provide a descriptive name like Xebium and copy the contents FitnesseSeleniumIDE.js into the text editor.
  • Now go ahead and record any test in seleniumIDE. When a test is finished, click the Source tab and copy the code in the into the copy buffer and paste it into the testpage created earlier. The only thing left is to change the localhost into url of the site you actually want to test against.
  • Save it, and run the FitNesse test.
